Situation Professionnelle

Déploiement de Postes Clients Windows & Jonction AD

AP — 1ère année BTS SIO SISR

Contexte

Dans un environnement de laboratoire virtualisé, j'ai simulé le rôle d'un technicien de support chargé de préparer une machine cliente vierge destinée à rejoindre l'infrastructure réseau administrée par un serveur de domaine de l'entreprise.

Besoin / Objectif

Le but était d'installer de manière conforme un système d'exploitation Windows 10/11 Pro, de configurer ses paramètres réseau de base de façon statique, et d'assurer son intégration sécurisée au sein d'un domaine Windows Server préexistant pour permettre une gestion centralisée.

Environnement technologique

  • Hyperviseur de virtualisation : Oracle VM VirtualBox / VMware Workstation
  • Système Client : Windows 10 / Windows 11 Professionnel
  • Système Serveur : Windows Server (Rôle AD DS & DNS installé)
  • Protocoles de base : TCP/IPv4, DNS d'infrastructure

Réalisation / Démarche

  • Étape 1 : Création de la machine virtuelle, allocation des ressources et installation complète de l'OS client Windows.
  • Étape 2 : Configuration des paramètres de la carte réseau virtuelle (attribution IP fixe et pointage DNS obligatoire vers l'IP du contrôleur de domaine).
  • Étape 3 : Modification des propriétés système de la machine pour modifier le groupe de travail par défaut et intégrer le nom de domaine de l'organisation.
  • Tests et validation : Authentification réussie à l'aide d'un compte utilisateur créé sur l'Active Directory et vérification de la création de l'objet ordinateur dans l'inventaire du serveur.

Bilan / Conclusion

Cet exercice a consolidé ma compréhension du fonctionnement de la résolution de noms (DNS) dans un environnement Microsoft Microsoft, élément indispensable pour qu'une jonction de domaine réussisse. L'atelier s'est déroulé sans encombre majeure grâce au respect rigoureux de l'adressage IP défini.

Compétences du référentiel

  • B1.4 — Option SISR : Administration des éléments de l'infrastructure système
