Situation Professionnelle 1

Mise en place d’un serveur Linux et sécurisation SSH.

Contexte

L’entreprise souhaitait déployer un nouveau serveur Linux destiné à héberger un service interne. Le serveur devait être sécurisé, accessible uniquement par SSH, et intégré à la supervision. J’ai été chargé de l’installation, du durcissement et de la configuration du serveur.

Objectifs

  • Installer un serveur Linux (Debian/Ubuntu)
  • Sécuriser l’accès SSH
  • Configurer le pare-feu (UFW)
  • Mettre en place un utilisateur administrateur
  • Installer les mises à jour et paquets nécessaires
  • Intégrer le serveur à l’outil de supervision

Travail réalisé

  • Installation du système Linux + configuration réseau
  • Création d’un utilisateur admin avec sudo sécurisé
  • Configuration SSH :
    • Désactivation du login root
    • Changement du port SSH
    • Authentification par clé SSH
  • Configuration du pare-feu UFW (SSH, HTTP, supervision)
  • Installation des outils : fail2ban, htop, net-tools
  • Ajout du serveur dans Zabbix (agent + monitoring)
  • Tests de sécurité et validation avec l’administrateur

Compétences validées

  • Administration Linux
  • Sécurisation SSH
  • Gestion des utilisateurs et permissions
  • Configuration réseau et firewall
  • Supervision Zabbix
  • Documentation technique
